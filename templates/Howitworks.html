How it works
Non technical explanation
The lineup optimizer uses the latest projections on injuries, opponents' status, expected goals, expected cards, etc. to identify which combination of players would produce the greatest production in fantasy points at the lowest cost.

In other words, the lineup optimizer saves you time by eliminating hours of research each night looking up stats, news, player situations, and trends by using this data to create an optimal lineup.

The projections we use in the Optimizer are constantly being updated; we then use those player and team projections to create the best possible lineup using the budget available for the given team.

If you are not satisfied with the suggested team, you can select a team that you want to avoid when optimizing the suggested lineup (i.e. assuming that you believe that Liverpool will most certainly lose in the coming round, then you can exclude all Liverpool players from the optimization).
Technical explanation
The lineup optimizer is using a three-step approach described below.

  • We collect and predict projections for all teams and players in the premier league. These projections will use historical data to predict future values e.g., change for a win, a goal, etc. These projections are based on various machine learning algorithms (e.g. Random Forest, gradient boosted trees, Naural networks, etc.), to increase the accuracy of the models as much as possible.

  • We transform the projected values, into projected points for the relevant player by using the rules defined. As we are using expected points for the next 6 rounds, we will discount future values, to take into account the increased uncertainty for injuries, quarantine, etc.

  • By editing the current team and the relevant budget, we now have the optimization problem which is to optimize the expected points in the coming round given the relevant constraints (budget, 11 players on the team, max 5 midfielders, etc.). This problem is used for solving a mathematical optimization problem. As this problem will have many local maximum, we need to search through all these local maxima to find the only global one.

    • Notice, that you can define new constraints, by excluding teams and players from the optimization, which is similar to adding constraints to the optimization problem.
